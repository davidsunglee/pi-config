### Issues

#### Critical (Must Fix)

**1. Evidence-block shape mismatch between SKILL.md and verifier.md.**
- `agent/skills/execute-plan/SKILL.md:609` — orchestrator emits blocks labeled `[Evidence 1]`, `[Evidence 2]`, …
- `agent/agents/verifier.md:19-27` — verifier contract shows blocks with header `[Criterion N] command: <exact command>`
- `agent/skills/execute-plan/verify-task-prompt.md:13-19` — template is silent on block header shape
- Issue: Orchestrator emits `[Evidence N]` while verifier expects `[Criterion N] command:`. The verifier's `evidence: <block number>` citation references won't align. Runtime effect: verifier will flag "insufficient evidence" on every file-inspection criterion even when evidence is present.
- Fix: Pick one shape and make SKILL.md, verify-task-prompt.md, and verifier.md all agree verbatim. Recommend `[Evidence for Criterion N]` or simply standardize on `[Criterion N]` throughout.

#### Important (Should Fix)

**2. `verify-task-prompt.md` doesn't pin the evidence-block format.**
- File: `agent/skills/execute-plan/verify-task-prompt.md:13-19`
- Issue: Says only "orchestrator has already executed every command-style Verify: recipe." Does not pin block header shape, numbering, or field order. Template is the single source of truth for what the verifier sees — without a pinned shape here, drift is easy.
- Fix: Inline the canonical block template (matching whichever shape is chosen in issue #1) into verify-task-prompt.md.

**3. "Modified Files" scope rule has an unspecified failure mode for auxiliary files.**
- File: `agent/agents/verifier.md:36`
- Issue: Rule "Do NOT read files outside `## Modified Files` unless a Verify: recipe explicitly names them" can cause silent FAIL when a recipe is written informally and doesn't name an unmodified auxiliary file needed for verification.
- Fix: Add a sentence to planner.md's "Format Constraints" reminding plan authors that any file needed for verification must be named in the recipe, AND clarify in verifier.md what to do when a recipe implies an unmodified file not in the named list.

**4. `{DIFF_CONTEXT}` has no size bounding.**
- File: `agent/skills/execute-plan/SKILL.md` (Step 10.2) and `verify-task-prompt.md`
- Issue: `git diff HEAD -- <modified files>` can produce very large output (e.g., new-file creations with large fixtures). SKILL.md Step 10.1 bounds command-style evidence (200 lines/20 KB), but diff context has no bound. A 10,000-line diff blows up verifier context.
- Fix: Apply the same truncation rule to `{DIFF_CONTEXT}` and document it in both SKILL.md Step 10.2 and verify-task-prompt.md.

**5. Protocol-violation re-dispatch for untyped concerns not explicitly counted against retry budget.**
- File: `agent/skills/execute-plan/SKILL.md`, Step 9 untyped-concerns re-dispatch
- Issue: An untyped-concerns task is re-dispatched once to the same model with a reminder — this re-dispatch appears to happen outside the Step 12 shared retry counter as worded. The spec is silent on whether this consumes a retry. Ambiguity could cause different readers to interpret it differently, and a strict reading that it does count would unexpectedly bite into budget.
- Fix: Add an explicit statement: "The protocol-violation re-dispatch for untyped concerns does NOT count against the Step 12 shared retry counter; it is a spec-compliance correction, not a task retry."

#### Minor (Nice to Have)

6. README.md "up to 8 tasks per wave" — number should reference MAX_PARALLEL_TASKS to avoid drift.
7. verifier.md:18 — numbering origin (1-based, in plan order) not stated explicitly.
8. Step 15 final-gate menu uses `(a)` and `(c)` (skips `(b)`) — should document the skip as intentional.
9. Step 9.7 §4 exhausted-budget path repeats Step 9.5 §5 logic verbatim — could use a cross-reference.
