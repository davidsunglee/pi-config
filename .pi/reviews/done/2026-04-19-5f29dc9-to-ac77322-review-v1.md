#### Important (Should Fix)

**1. Step 15 §0 numbering conflict — two `### 1` sections.**
- File: `agent/skills/execute-plan/SKILL.md`, Step 15
- Issue: A new `### 0. Deferred integration regression gate` is prepended while the existing structure uses `### 1.`, `### 2.`, etc. A `### 0` subsection is unusual and inconsistent with the rest of the document; prose references "Steps ### 1. Move plan to done, ### 2. Close linked todo, and ### 4. Branch completion" which reads awkwardly.
- Why it matters: LLM orchestrators follow headers literally; odd numbering increases the chance of a model skipping §0 on the assumption that "numbered steps start at 1."
- How to fix: Renumber as `### 1. Deferred integration regression gate` and shift subsequent subsections up by 1, or promote the gate to a named subsection without a number (e.g., `### Deferred regression gate`).

**2. `tools:` field uses lowercase — may fail to map to real tool names.**
- File: `agent/agents/verifier.md:5` — `tools: read, grep, find, ls`
- Issue: Other subagent definitions in this repo typically use capitalized tool names (`Read`, `Grep`, `Glob`, `Bash`). If the harness matches by exact name, `read`/`grep`/`find`/`ls` will not map to the real tools and the verifier will have no file access — returning all FAILs for file-inspection criteria.
- Additionally: `find` and `ls` are not first-class Claude tools — they're Bash commands. The canonical file-listing tool is `Glob`.
- How to fix: Capitalize to `Read, Grep, Glob` (replacing `find`/`ls` with `Glob`). Also fix prose in verifier.md that says "use the `read`, `grep`, `find`, `ls` tools."

**3. `git diff HEAD` assumption may be wrong for split-wave/sub-task scenarios.**
- File: `agent/skills/execute-plan/SKILL.md`, Step 10.2, `{DIFF_CONTEXT}` bullet
- Issue: The spec says "Do NOT substitute a committed-range diff" — which is too absolute if a sub-task (from Step 9.5 §5 `(s) Split into sub-tasks`) was dispatched after a wave commit. Sub-task changes may be committed before Step 10 runs, making `git diff HEAD` return an empty diff and causing the verifier to return `FAIL: insufficient evidence` for all file-inspection criteria.
- How to fix: Add an explicit carve-out or assertion that sub-task dispatches are always pre-commit; or specify a fallback diff range for committed sub-tasks.

**4. Retry budget double-counting underspecified across Step 9.5 and Step 9.7.**
- File: `agent/skills/execute-plan/SKILL.md`, Steps 9.5 §5 and 9.7 §4
- Issue: Both sections say their re-dispatch "counts against the Step 12 retry budget" — but if a task bounces between both gates (BLOCKED → re-dispatch → done_with_concerns → re-dispatch), the budget arithmetic is not specified for the mixed case.
- How to fix: Add one sentence in Step 12: "All re-dispatches from Step 9.5, Step 9.7, and Step 10 failure routing share a single per-task retry counter. Exhaustion in one path exhausts it for all paths."
