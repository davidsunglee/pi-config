## Problem

The plan review step in generate-plan (Step 3.5) dispatches a `plan-executor` agent with the `plan-reviewer.md` template. The plan-executor system prompt is oriented toward executing tasks (status codes, file output conventions, escalation rules) — none of which are relevant to reviewing a plan for structural issues.

The existing `plan-reviewer.md` agent file at `agent/agents/plan-reviewer.md` is only 8 lines and delegates entirely to the task prompt, making the agent definition a no-op wrapper.

## Goal

Create a dedicated `plan-reviewer` agent with a system prompt tuned for plan review: understanding plan structure, checking spec coverage, validating dependencies, assessing task sizing, and outputting the structured review format (Approved / Issues Found with Error/Warning/Suggestion severity).

This parallels the work in TODO-efc9f70b where we're creating a dedicated `code-reviewer` agent to replace plan-executor-as-reviewer for code review.

## Also: Migrate generate-plan to model matrix

Generate-plan currently reads `modelTiers` from `~/.pi/agent/settings.json`. As part of the model matrix extraction (moving model config to `~/.pi/agent/models.json`), generate-plan should be updated to read from the new location. This affects:

- Step 3 (generate-plan SKILL.md): model selection for the plan-generator subagent
- Step 3.5: model selection for the plan-reviewer subagent
- Any references to `modelTiers` in settings.json within the skill
