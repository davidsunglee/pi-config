Follow-up to TODO-bbe2c2ad. That todo handles low-risk extractions and local cleanups in `agent/skills/execute-plan/SKILL.md`. This todo covers three structural consolidations that were deferred because they restructure section boundaries and warrant their own scoped change.

## Goal
Further reduce `agent/skills/execute-plan/SKILL.md` by collapsing two near-duplicate gates into one, removing cross-restated worktree-reuse logic, and centralizing the retry-budget rule — without changing observable behavior.

## Scope (committed)
Three items.

### Item A: merge Steps 9.5 and 9.7 into one "wave gate" step

Steps 9.5 (blocked-task escalation) and 9.7 (wave-level concerns checkpoint) share the same shape: drain the wave → collect a set of tasks matching some condition → present a single combined view → ask for a per-task choice → re-dispatch and re-enter the loop. The only differences are:

- Trigger condition: `STATUS: BLOCKED` (9.5) vs. `STATUS: DONE_WITH_CONCERNS` (9.7).
- Per-task menu options: `(c) more context / (m) better model / (s) split into sub-tasks / (x) stop` for blocked, vs. `(c) continue to verification / (r) remediate selected / (x) stop` for concerned.
- Whether the wave can exit the gate with the condition still attached: 9.5 must clear all BLOCKED tasks (no skip), 9.7 can exit with `(c) Continue` leaving DONE_WITH_CONCERNS in place for the verifier.

Replace both with a single "wave gate" step that defines the shared shape once, then specifies the two cases (blocked / concerned) in a small parameter table. Preserve all current semantics including the gate-exit rules and the control-flow ordering "blocked-handling before concerns-handling before verification."

### Item B: trim Step 0 + Step 3 cross-restated worktree-reuse logic

The `(n) Create a new worktree instead` fallback path, the reuse-log mandatoriness rule, and the customizability rules for the workspace setting are each stated 2–3 times across Step 0 and Step 3's defaults table + paragraphs. Pick one home (Step 0) for the full reuse decision logic; have Step 3 just present the settings UI and reference Step 0 for the underlying rules.

### Item C: de-duplicate the retry-budget rule across Steps 9.5, 9.7, and 12

The shared-counter rule (3 retries per task across all re-dispatch paths, plus the sub-task-split-doesn't-bypass clause) is currently restated in Step 9.5 §5, Step 9.7 §3, and Step 12. Keep the canonical statement in Step 12 and replace the other two with a one-line pointer.

Note: this item depends on item A (the merged wave gate) since the canonical "Step 9.5 §5" and "Step 9.7 §3" anchors disappear after the merge. Plan order should run A before C.
