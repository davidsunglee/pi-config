## Problem

`agent/skills/refine-code/SKILL.md` dispatches `code-refiner` using the `standard` model tier. Current `model-tiers.json` maps `standard` to an Anthropic model, and `dispatch.anthropic` maps to `claude`.

`code-refiner` is a coordinator: it is expected to call `subagent_run_serial` to spawn `code-reviewer` and `coder` agents. A Claude CLI child does not have pi's custom orchestration tools, so the coordinator may be launched in an environment where it cannot perform its core job.

## Recommended direction

Ensure coordinator agents that need pi orchestration tools dispatch via `pi`, regardless of model tier.

Options to evaluate:

- Use a pi-backed coordinator tier, e.g. `crossProvider.standard`.
- Add an explicit `coordinator` tier to `model-tiers.json`.
- Special-case `refine-code` so `code-refiner` always gets `cli: "pi"`.

## Acceptance criteria

- `refine-code` dispatches `code-refiner` through a CLI that exposes `subagent_run_serial`.
- The model/CLI resolution rule is documented in the skill and remains consistent with `model-tiers.json`.
- Any tests or contract checks needed to prevent regression are added or updated.
