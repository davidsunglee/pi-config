## Resolution

**Addressed by in-place reform rather than excision.** HEAD (commit `7fe3eb5` and the surrounding simplification series) kept Step 9.7 as a gate but stripped the high-complexity parts this todo flagged. Closing as done — the original cut-it-entirely prescription is no longer the right move.

### What HEAD looks like (vs v2 @ `14a89882`)

- **Step 9.7 reformed in place.** Typed routing (correctness/scope/observation), severity ordering, per-type menus, and the acknowledge-and-continue rules are gone. HEAD's 9.7 is a single combined view of concerned tasks with three options: `(c)` continue to verification / `(r)` remediate selected tasks with user guidance / `(x)` stop. No `Type:` protocol, no `## Concerns To Address` ceremony, no forced re-dispatch on correctness/scope.
- **Three-set integration model consolidated into Step 7.** Step 7 now hosts the canonical "Integration regression model" subsection; Step 11 and Step 15 reference it instead of restating.
- **Step 10 verifier prose compacted** (wave 3 of the simplification plan).
- **Everything else kept:** verifier subagent with per-criterion PASS/FAIL, identifier-based test tracking, Step 15 final integration gate, shared retry counter, no-skip rule, deferred integration regressions.

Diff v2 → HEAD: 144 insertions / 152 deletions. Net line count barely changed (1008 → 1000) because review-iteration commits added hardening elsewhere (sub-task budget rule, `MODIFIED_FILES` assembly, truncation specifics, final-gate strictness) that offset most of the savings from 9.7 and the Step 11 consolidation. Complexity *per step* dropped even though the total line count did not.

### Why not still cut 9.7

The original reasoning said 9.7's value was low *because v2's version was high-complexity ceremony*. HEAD's reformed 9.7 changes that ratio — it's now a cheap gate with a clear purpose:

- Surfaces worker-flagged concerns to the user *before* verifier budget is spent on potentially-questionable work.
- `(r)` gives explicit remediation without forcing `(x)`.
- Combined-view format respects the "don't interrupt each worker as it returns" principle.

Cutting it now would save ~50 lines of already-simplified prose at the cost of losing the pre-verification user checkpoint. Net negative.

### Leftover / related

- **Deferred integration regressions feature** was flagged in the original todo as "borderline — leave for now." Still untouched at HEAD and still borderline. If complexity keeps biting, revisit separately — dropping it collapses the three-set model to two sets but touches Steps 11, 12, 13, and 15. Not worth opening a new todo until there's concrete evidence of pain.
- **Spec/plan alignment:** the hardening spec and plan (commits `ec219f8`, `5e82b0e`) described the *additions*. The simplification series that followed (`3ef54e1` onward) did the in-place reform without a paired spec. If future work revisits these trade-offs, the simplification rationale lives in this todo and in the commit messages; there is no standing spec to reconcile against.
