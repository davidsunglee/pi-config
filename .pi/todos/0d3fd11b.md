## Problem

The handoff from reviewer agents to refiner agents should not depend on large inline reviewer output being transported through orchestration result messages. The `plan-reviewer` → `plan-refiner` and `code-reviewer` → `code-refiner` paths need an explicit artifact-based contract owned by the reviewer.

## Desired change

Update the reviewer/refiner workflow so:

1. `plan-reviewer` writes the full plan review to a designated artifact path supplied in its task prompt.
2. `plan-reviewer` returns a compact response to `plan-refiner` containing the path to the review artifact it wrote.
3. `plan-refiner` reads that artifact path and uses the file contents as the authoritative full review.
4. `code-reviewer` writes the full code review to a designated artifact path supplied in its task prompt.
5. `code-reviewer` returns a compact response to `code-refiner` containing the path to the review artifact it wrote.
6. `code-refiner` reads that artifact path and uses the file contents as the authoritative full review.

## Affected areas

- `agent/agents/plan-reviewer.md`
- `agent/agents/code-reviewer.md`
- `agent/skills/refine-plan/refine-plan-prompt.md`
- `agent/skills/refine-code/refine-code-prompt.md`
- Potentially `agent/skills/refine-plan/SKILL.md` and `agent/skills/refine-code/SKILL.md` if outer result parsing needs to document or validate the new contract.

## Requirements

- The refiner prompts must provide each reviewer with a concrete output artifact path before dispatch.
- Reviewers must write the complete review body to that path using their write capability.
- Reviewer final responses should be compact and include the artifact path, not the full review text.
- Refiners must fail clearly if the reviewer response does not contain a readable artifact path or the artifact file is empty.
- Preserve existing provenance requirements for persisted reviews, including `**Reviewer:** <provider>/<model> via <cli>` at the start of review artifacts.
- Preserve the rule that refiners do not perform their own reviews; they only consume reviewer-authored artifacts.
- Apply this contract consistently across first-pass review, re-review, and final verification flows.

## Acceptance criteria

- `plan-reviewer` can complete by writing the full plan review to the supplied path and returning only a compact artifact-path handoff.
- `plan-refiner` reads the reviewer-written artifact and persists/uses the full review content without relying on inline `finalMessage` review text.
- `code-reviewer` can complete by writing the full code review to the supplied path and returning only a compact artifact-path handoff.
- `code-refiner` reads the reviewer-written artifact and persists/uses the full review content without relying on inline `finalMessage` review text.
- Large reviews no longer get truncated during reviewer/refiner handoff.
- Manual smoke runs for both refine-plan and refine-code confirm the refiner receives only the artifact path from the reviewer and reads the full artifact from disk.
