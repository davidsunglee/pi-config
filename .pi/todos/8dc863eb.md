## Observation

The workflow's complexity is almost entirely carried by prompt instructions rather than code. execute-plan alone is a 13-step state machine described in ~300 lines of prose. Every execution depends on the orchestrating agent perfectly following procedural instructions — model tier resolution, wave computation, template filling, retry logic, cross-provider selection, worktree setup delegation, commit management, and state tracking.

This works today because it runs on opus-class models that are good at following complex instructions. But it's fragile in ways that won't be obvious until they bite — a skipped step, a hallucinated model name, a lost piece of state between waves.

## The Spectrum

There's a spectrum of how much to move from prose → code:

### Minimal: Keep prose, add guardrails
- State sidecar file (TODO-307824c4) adds some durability
- Concrete model resolution (TODO-37d49fa8) removes one hallucination vector
- Commit-per-wave (TODO-13d7f8d6) adds checkpoints
- This is the "fix the worst fragilities" approach

### Middle: Pi extension for orchestration
- A `/execute-plan` extension that handles the mechanical parts in TypeScript:
  - Parse plan file, compute waves, resolve models from settings.json
  - Dispatch subagents with filled templates
  - Commit after each wave, run tests, track state
  - Present results in TUI
- The agent still makes judgment calls (handling BLOCKED, deciding retries)
- The extension handles bookkeeping

### Maximum: Purpose-built CLI tool
- Standalone tool that reads plan files and orchestrates execution
- Pi becomes just the executor (plan-executor subagent)
- Loses flexibility but gains reliability

## Recommendation

Start with "minimal" (the other todos in this batch). Evaluate after using the improved workflow for a few weeks. If the same class of failures keeps happening (skipped steps, wrong models, lost state), invest in the "middle" approach — a pi extension.

The "maximum" approach is probably overkill unless the workflow becomes a team-shared tool rather than a personal one.
