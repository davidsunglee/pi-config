## Problem

Parallel plan execution produces cross-file inconsistencies that per-task verification can't catch. Example: the wiki overview's ToC linked to filenames that didn't match what the other workers actually created (`02_ast_nodes.md` instead of `02_model_layer.md`). Individual workers can't validate cross-file references because they only see their own output.

## Solution: Two pieces

### 1. Standalone `review` skill

A reusable skill that takes a file list or directory and dispatches a capable-model agent in a **clean context** to evaluate all outputs simultaneously.

**What it checks:**
- Cross-file references (do linked files exist? are names consistent?)
- Structural consistency (naming conventions, formatting patterns across files)
- Content gaps (sections promised but missing, stale references)
- Factual spot-checks (pick specific claims and verify against source)

**Output:** Structured findings report — each finding has severity (error/warning/note), file, line, description, and suggested fix.

**Must use a capable model in a clean context** — the reviewer needs to see all files without being biased by the generation process.

Reusable beyond plan execution: reviewing PRs, checking docs after manual edits, validating any multi-file output.

### 2. `review` option in execute-plan

After all waves complete (before moving plan to `done/`), if review is enabled:
1. Invoke the review skill on all created/modified files
2. If actionable findings exist, dispatch fix-up tasks
3. Only move to `done/` after review passes

**When to trigger:**
- Explicit `review: true` option
- Auto-detect: plans where all tasks are Create (new file generation), doc/wiki plans, plans with 5+ output files

### Design notes

- Review needs **all outputs visible simultaneously** — that's the key difference from per-task verification
- The execute-plan Step 9 (verify wave output) stays as-is for per-task acceptance criteria checks — review is a complementary cross-cutting pass
- Review skill should be invocable independently: `/skill:review docs/wiki/`

## Resolution\n\nSubsumed by `requesting-code-review` skill (ported from superpowers) with mandatory trigger in `execute-plan` after all waves complete (Step 12). The code-reviewer subagent dispatched with a capable-tier model in fresh context covers cross-file consistency, broken links, and structural issues as part of its standard review checklist.
