## Problem

The answer extension and todo extension are independent tools with no formal connection. During todo refinement (steps 4-6 of the workflow), the user uses `/answer` to respond to model questions, but the refined answers live in the conversation — they don't flow back into the todo body. The user has to manually edit the todo with insights from the Q&A loop.

## Changes

Options (pick one or combine):

### Option A: Answer extension auto-appends to todo
If the conversation is in the context of refining a specific todo (detectable by recent `todo get` calls or explicit user signal), the answer extension appends a `## Refinement` section to the todo body with the Q&A pairs.

### Option B: Explicit `/answer --todo TODO-xxx` flag
User explicitly tells the answer extension which todo to write back to. After submitting answers, the compiled Q&A is appended to the todo body.

### Option C: Keep it conversational, just make it easier
Add a "save to todo" action in the answer TUI that writes the current Q&A to a specified todo. Lightweight, no auto-detection needed.

## Considerations
- Option C is probably the right level of automation — explicit, no magic
- Need to decide format for appended content (raw Q&A pairs? summarized decisions?)
- The answer extension currently uses haiku for extraction — it doesn't need the todo tool itself, just needs to call it after
