## Goal

Reduce orchestrator context pollution in the planning pipeline by changing only the initial `generate-plan -> planner` handoff so file inputs are passed by path instead of embedding whole markdown documents into the planner prompt.

## Context

Today, `generate-plan` reads file inputs and injects their full contents into `{TASK_DESCRIPTION}` before dispatching the `planner` subagent. That keeps the planner prompt self-contained, but it repeatedly loads large specs, RFCs, and design docs into the orchestrator's own context window.

This task is intentionally limited to the first planning-stage dispatch. The later `generate-plan` review/edit loop is a separate follow-up (`TODO-58b1648b`). `execute-plan` and `execute-plan -> coder` are not part of this change.

## Requirements

- Change only the initial `generate-plan -> planner` dispatch.
- For **todo** and **freeform** inputs, keep the current inline behavior.
- For **all file inputs** (including `.pi/specs/...` and arbitrary design/RFC files), switch to path-based handoff for the planner.
- Do **not** inline the full file contents into the planner prompt for file inputs.
- For file inputs, the orchestrator may do a **bounded preamble read only** to extract metadata; it should not read and embed the full document body into the prompt.
- Apply strict provenance extraction rules to all file inputs:
  - inspect only the preamble area at the top of the file
  - only exact supported lines count
  - supported lines are:
    - `Source: TODO-<id>`
    - `Scout brief: .pi/briefs/<filename>`
- Add an explicit primary-artifact field to the planner handoff, e.g. `Task artifact: <path>`.
- Keep provenance inline when available:
  - `Source todo: TODO-<id>` if extracted
  - `Source spec: .pi/specs/<filename>` when the input path is under `.pi/specs/`
  - `Scout brief: .pi/briefs/<filename>` when extracted
- Update the planner dispatch contract so it explicitly tells the planner to read the handed-off artifact from disk before planning.
- If a scout brief path is provided, the planner contract must explicitly require the planner to read that brief as well.
- If a referenced scout brief file is missing, warn and continue without failing.
- For file-input planning runs, derive the output plan filename from the **input filename**, not from the full document body.

## Constraints

- Keep the `generate-plan` review/edit loop behavior unchanged in this task.
- Small internal refactors are allowed if they help isolate the new planner-dispatch path, but there should be no intended behavior change for `plan-reviewer` or planner-edit passes.
- Do not change `execute-plan` orchestration behavior.
- Do not change `execute-plan -> coder` prompts; they remain self-contained and task-scoped.
- Use plain filesystem paths as the handoff primitive; no markdown links or `file://` URIs are needed.
- Preserve current warning/continue behavior for missing scout briefs.
- Prefer lightweight provenance over snapshot hashing; no full-content fingerprinting is required.

## Acceptance Criteria

- `generate-plan` uses path-based handoff for the initial planner dispatch whenever the input is a file path.
- For file inputs, the planner prompt contains the artifact path and inline provenance metadata, but not the full file contents.
- For todo and freeform inputs, existing inline handoff behavior remains intact.
- Provenance extraction still works from exact supported lines in the top preamble of file inputs.
- A referenced scout brief path is passed through inline, and the planner contract explicitly instructs the planner to read it from disk.
- Missing scout briefs produce a warning and do not block planning.
- The planner agent/prompt documentation is updated so the file-based handoff contract is explicit and testable.
- The review/edit loop remains behaviorally unchanged.
- Verification is done with real smoke tests and prompt/transcript inspection sufficient to show:
  - file inputs no longer inline full document contents into the initial planner dispatch
  - todo/freeform inputs still behave as before
  - missing scout briefs warn and continue

## Non-Goals

- Moving `generate-plan -> plan-reviewer` to file-based handoff
- Moving planner edit passes to file-based handoff
- Changing review/approval semantics in the planning loop
- Changing `execute-plan` or execution-worker prompt scope
- Designing a generic snapshot/fingerprinting system for planning artifacts
