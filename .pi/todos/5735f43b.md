## Problem

When generate-plan creates a plan from a todo, nothing ties them together. Closing the todo after plan completion is manual — though capable models often do it anyway.

## Solution

One-way link: the plan header carries the source todo ID. Execute-plan uses it to auto-close.

### generate-plan: add source todo ID to plan header
When generating a plan from a todo, include `**Source:** TODO-<id>` as an inline field after the Tech stack section in the plan header. Only added when the plan originates from a todo.

### execute-plan: close linked todo on completion
In Step 13, after moving the plan to `done/`, check the plan header for a `**Source:** TODO-<id>` line. If present:
1. Set the todo status to "done"
2. Append to the todo body: `Completed via plan: .pi/plans/done/<plan-filename>.md`
3. Report in the completion summary (e.g., "Closed TODO-5735f43b")

Skip silently if:
- No `Source:` line in the plan header
- The referenced todo is already closed
- The todo ID doesn't exist

## Files
- Modify: `~/.pi/agent/agents/plan-generator.md` — add `**Source:** TODO-<id>` after Tech stack when source is a todo
- Modify: `~/.pi/agent/skills/execute-plan/SKILL.md` — Step 13, check for source todo, close it, and append completion note
