## Problem

execute-plan Step 11 says "leave the plan file so it can be resumed" but there's no actual resume mechanism. The plan file has no state tracking — no record of which tasks completed, which failed, what wave you were on. Restarting after a partial execution would re-run everything from wave 1.

## Changes

### State sidecar file
Write a `.pi/plans/<plan-name>.state.json` alongside the plan file, tracking:
```json
{
  "plan": "2026-04-06-my-feature.md",
  "started_at": "...",
  "current_wave": 3,
  "completed_tasks": [1, 2, 3, 4],
  "failed_tasks": [5],
  "skipped_tasks": [],
  "pre_execution_sha": "abc123",
  "wave_commits": {
    "1": "def456",
    "2": "ghi789"
  }
}
```

Update after each wave completes.

### Resume on startup
At the start of execute-plan (between Step 1 and Step 2), check for an existing `.state.json`. If found, offer:
- **Resume** from last completed wave
- **Restart** from scratch (delete state file)

- Modify: `~/.pi/agent/skills/execute-plan/SKILL.md` — add state tracking after each wave, add resume check at startup
