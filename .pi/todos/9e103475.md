Goal:

Rename the current working-message extension and update the streaming working message to use the rainbow-editor effect in a constrained way.

Scope:

- Rename `agent/extensions/whimsical.ts` to `agent/extensions/working-message.ts`.
- Update any in-repo references or docs affected by that rename.
- Keep ownership of working-message behavior in this extension.
- Do not change `agent/extensions/working-indicator.ts`.

Behavior requirements:

- Keep the existing whimsical message list unchanged.
- Keep the current selection behavior: choose one random working message per turn.
- Apply an animated shine effect to the entire working message whenever the working message is visible.
- When the extension receives `thinking_start`, switch the entire working message to the rainbow palette + shine effect together.
- When the extension receives `thinking_end`, return the message to shine-only.
- If the model/provider never emits thinking events for a turn, the message should remain shine-only for that entire turn.

Implementation guidance:

- Start from the rainbow rendering approach used in the `rainbow-editor` example:
  https://github.com/badlogic/pi-mono/blob/main/packages/coding-agent/examples/extensions/rainbow-editor.ts
- Use the example palette/animation as-is initially.
- Limit changes to the working message; do not change unrelated UI components.

Fallback/readability requirements:

- If the UI context cannot support the effect cleanly, fall back to the original unstyled whimsical message.
- Target readability in both dark and light themes, but keep the example palette exactly for the initial implementation.
- Avoid performance regressions.

Verification:

- Verify the behavior visually in theme `nord`.
- Verify the behavior visually in the built-in `light` theme.
- Confirm normal turn behavior, shine-only behavior, and the transition into and out of rainbow+shine on `thinking_start` / `thinking_end`.
