## Summary

Follow up after the planner-dispatch handoff change by reducing orchestrator context pollution in the `generate-plan` review/edit loop as well.

The first slice only changes `generate-plan -> planner` so file inputs are handed off by path instead of embedding full markdown contents. This follow-up should evaluate and, if appropriate, implement the same general optimization for the later planning-stage passes:

- `generate-plan -> plan-reviewer`
- planner edit pass used after plan review findings

## Motivation

Even if the initial planner dispatch becomes path-based, the review/edit loop can still re-inline large artifacts:

- the generated plan contents
- the original spec/design-doc contents
- review findings plus surrounding context

That means the orchestrator may still repeatedly load large markdown documents into its own context during iterative planning, reducing some but not all of the token/context savings.

## Goal

Design and implement a file-based handoff contract for the `generate-plan` review/edit loop that preserves correctness and review quality while reducing repeated markdown inlining in the orchestrator.

## Questions to resolve

1. Should `plan-reviewer` receive the plan and source artifact by path instead of full inline contents?
2. Should the planner edit pass read the current plan from disk instead of receiving the full plan inline?
3. What provenance and snapshot-safety metadata should remain inline when using path-based handoff?
4. Should review findings remain inline even if plan/source artifacts move to paths?
5. Does `plan-reviewer` need an explicit contract update stating that it must read the handed-off files from disk?
6. Are there any cases where keeping inline snapshot semantics is still preferable for the edit pass?

## Initial constraints

- This is a follow-up to the planner-only handoff optimization.
- Keep `execute-plan` and `execute-plan -> coder` out of scope.
- Preserve review quality and approval semantics.
- Avoid widening worker scope unnecessarily.

## Suggested completion criteria

- Decide whether `plan-reviewer` should switch to file-based handoff
- Decide whether the planner edit pass should switch to file-based handoff
- Specify what stays inline vs by path for each pass
- Specify any provenance/snapshot guardrails needed
- If implemented, verify behavior with smoke tests and prompt/transcript inspection
