## Summary
`refine-code` currently launches the `code-refiner` coordinator using the dispatch resolved from the `standard` tier. When that resolves to `claude`, nested reviewer/remediator calls can get coerced into Claude-only tier labels (`opus`/`sonnet`/`haiku`) instead of preserving concrete provider/model ids from the model matrix.

## Problem
In a real run reviewing `5f29dc9..ac77322`, the `code-refiner` prompt asked for cross-provider reviews with `openai-codex/gpt-5.4`, but the nested runtime reported that only `sonnet`, `opus`, or `haiku` were accepted and substituted `opus`. That meant the intended cross-provider reviews did not actually run.

## Narrow fix
Update the `refine-code` skill so the top-level `code-refiner` coordinator is always dispatched via `pi`, while leaving its nested reviewer/coder routing driven by the model matrix.

## Expected outcome
- `code-refiner` still runs on the configured coordinator model.
- Nested `code-reviewer` calls can preserve `openai-codex/gpt-5.4` for cross-provider passes.
- Nested `coder` calls can preserve the exact Anthropic model ids from the matrix.
