## Summary

Add a dedicated `sessionName` field to the footer's `FooterColors` type so the session name is visually distinct from `pwd` and `branch`. Also adjust three default token fallbacks to better semantic colors.

## Changes

### 1. New `sessionName` field

Add `sessionName` to the `FooterColors` type and use it when rendering the session name (currently shares `pwd` color).

**Per-theme overrides (purple/magenta family):**

| Theme | Color | Hex | Notes |
|---|---|---|---|
| carbonfox | magenta | `#be95ff` | Soft lavender — palette accent |
| everblush | magenta | `#c47fd5` | Violet — palette accent |
| nord-dark | nord15 | `#b48ead` | Mauve — classic Nord purple |

**Default fallback token:** `warning`
- Warm gold/yellow in most themes, distinct from `error` (pwd) and `success` (branch).
- Carbonfox is an outlier (teal), but its per-theme override covers it.

### 2. Adjust default token fallbacks

| Field | Current default | New default | Rationale |
|---|---|---|---|
| `pwd` | `dim` | `error` | Red/pink across themes — brighter, more visible path |
| `tokens` | `muted` (gray) | `border` | Blue across all three themes — subtle blue for token counts |
| `contextUsage` | `muted` (gray) | `accent` | Bright blue/cyan across themes — prominent context percentage |

### 3. Files to change

- `agent/extensions/footer.ts`:
  - Add `sessionName` to `FooterColors` type
  - Add `sessionName` to each entry in `THEME_COLORS` (carbonfox, everblush, nord-dark)
  - Add `sessionName` to `DEFAULT_TOKENS` with value `"warning"`
  - Update `DEFAULT_TOKENS.pwd` from `"dim"` to `"error"`
  - Update `DEFAULT_TOKENS.tokens` from `"muted"` to `"border"`
  - Update `DEFAULT_TOKENS.contextUsage` from `"muted"` to `"accent"`
  - Change the session name render line from `colorize("pwd", sessionName)` to `colorize("sessionName", sessionName)`

## Design notes

- Purple/magenta was chosen for `sessionName` because it's present in all three palettes, plays a consistent "highlight" role, and is clearly distinct from both green (branch) and red (pwd).
- The `warning` fallback for `sessionName` was preferred over `accent` for its warmer tone in most themes. The carbonfox teal outlier is handled by the per-theme override.
- `error` for pwd is semantically imperfect but is the only token that consistently resolves to red/orange across themes. Acceptable for now.
