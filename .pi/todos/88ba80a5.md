## Goal

Ensure `execute-plan` reliably distinguishes pre-existing integration-test failures from new regressions by using stable, suite-native test identifiers throughout baseline, post-wave, and final-gate reconciliation.

## Context

`execute-plan` relies on the `test-runner` subagent and failing-identifier sets to decide whether a failure was already present at baseline or was introduced by plan execution. If identifiers are unstable, synthesized, normalized inconsistently, or extracted differently across phases, the reconciliation model can misclassify existing failures as new regressions or miss real regressions.

## Acceptance Criteria

- Audit `agent/skills/execute-plan/` and `agent/agents/test-runner.md` identifier-extraction guidance.
- Make the identifier contract precise enough for common runners (`go test`, `pytest`, `cargo test`, Jest/Vitest, crashes before test names).
- Ensure baseline, wave, debugger re-test, and final-gate paths all consume the same artifact format and compare identifiers byte-for-byte.
- Add or update regression coverage/docs so future changes do not reintroduce unstable identifier extraction.
- Explicitly document behavior when no stable per-test identifier is available.
