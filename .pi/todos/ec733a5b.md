## Changes

### 1. Reorder footer fields

Current field order puts tokens and cost before context usage. Reorder to:

1. Context usage
2. Tokens
3. Cost

Context usage is the most operationally relevant stat (tells you when you're approaching compaction), so it should come first.

### 2. Configurable color for cost sub-indicator

The cost field can show a `(sub)` indicator for subagent cost. Currently this inherits the cost field's color. Make the sub-indicator color independently configurable in the theme's footer color overrides, so it can be dimmed or styled differently from the main cost value.

### 3. Consider moving session name to prompt bar

Claude Code shows the session name in the prompt bar rather than the footer. Evaluate moving the session name out of the footer and into the prompt bar:

- Frees footer space for stats
- Prompt bar is a more natural location (session identity vs. session metrics)
- Check if pi's extension API supports writing to the prompt bar area — if not, this may not be feasible

## Completion criteria

- Footer fields appear in order: context usage, tokens, cost
- Cost sub-indicator color is independently configurable via theme footer overrides
- Session name placement decision is made and implemented (either moved to prompt bar or kept in footer with rationale)
