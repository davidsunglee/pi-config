## Problem

The review extension (`~/.pi/agent/extensions/review.ts`) is 67KB of code implementing a standalone review system with session management, loop fixing, and custom instructions. It doesn't integrate with the plan pipeline — the workflow uses `requesting-code-review` skill + `code-reviewer.md` dispatched as a subagent instead.

Features like loop fixing and persistent custom review instructions aren't used. Review sessions (context forking) is an interesting concept tracked separately in TODO-b3c7023c. No projects use `REVIEW_GUIDELINES.md`.

## Changes

### 1. Verify the extension is self-contained
Before deleting, confirm no other code depends on it:
- `grep -r "review" ~/.pi/agent/extensions/*.ts` — check if other extensions import from or reference `review.ts`
- Check for persisted state keys (the extension uses `REVIEW_STATE_TYPE`, `REVIEW_ANCHOR_TYPE`, `REVIEW_SETTINGS_TYPE`) — verify no other extension reads these
- Check skill files for any references to `/review` command

### 2. Delete the file
Delete `~/.pi/agent/extensions/review.ts`.

### 3. Verify clean startup
Start a fresh pi session and confirm:
- No errors on startup related to missing review extension
- `/review` command is no longer registered
- Other extensions load normally
