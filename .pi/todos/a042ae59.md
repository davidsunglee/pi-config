## Goal
Add a `"nord-dark"` entry to `THEME_COLORS` in `agent/extensions/footer.ts` to give the footer custom colors using the Nord palette.

## Color Mapping

| Field          | Nord color   | Hex       | Rationale                                          |
|----------------|--------------|-----------|----------------------------------------------------|
| `pwd`          | nord12       | `#d08770` | Orange — "orange/red vein". Session name shares this color (no separate field). |
| `branch`       | nord14       | `#a3be8c` | Green — keep default, matches git convention       |
| `tokens`       | nord9        | `#81a1c1` | Subtle/muted blue, similar to modelName but softer |
| `modelName`    | nord8        | `#88c0d0` | Accent blue — slightly brighter than tokens        |
| `contextUsage` | nord8        | `#88c0d0` | "Slightly brighter blue" than tokens               |
| `contextWindow`| nord4        | `#d8dee9` | Silver / bright gray                               |
| `cache`        | nord3        | `#4c566a` | Muted/dim — complementary, keeps cache subtle      |
| `cost`         | nord13       | `#ebcb8b` | Yellow/gold — matches carbonfox/everblush approach  |
| `symbols`      | nord3        | `#4c566a` | Muted — separators and punctuation stay subdued     |
| `statuses`     | nord3        | `#4c566a` | Muted — extension status line stays subdued         |

## Implementation
Add a single block to the `THEME_COLORS` object in `agent/extensions/footer.ts`, following the existing `carbonfox` and `everblush` entry style (with inline hex comments).

## Notes
- All colors come from the existing Nord palette (`nord0`–`nord15`) — no off-palette hex values needed.
- Session name inherits `pwd` color (orange) since `FooterColors` has no separate field for it — same as carbonfox and everblush.
- Context usage escalation colors (>70% warning, >90% error) are handled by the theme itself and are not overridden here.
