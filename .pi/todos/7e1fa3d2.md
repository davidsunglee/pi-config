## 1. Add `validate-plan` tool (schema gate at seam)

**Problem:** Execute-plan parses conventions the planner is never required to produce: the `Dependencies` section format ("Task X depends on: Y, Z"), the `**Source:** TODO-<id>` line, per-task `**Files:**` and model recommendation. Malformed output silently breaks wave ordering or skips todo auto-close.

**Scope:**
- Define a plan schema (required sections, per-task shape, dependency line format)
- Build a validator (script or small tool) that reports structural defects
- Wire into generate-plan exit gate (before handoff) and execute-plan Step 2 (before execution)
- Shared source of truth; both skills call the same validator

**Size:** small

---

## 2. Require machine-checkable acceptance criteria

**Problem:** AC today is free-form English. Step 10 verification is orchestrator self-audit reading files "point-by-point" — the weakest gate in the system.

**Scope:**
- Update spec + plan format so each AC must be one of: (a) a test the coder must add, (b) a command whose exit code is the gate, (c) a file-content assertion (regex / presence / shape)
- Update define-spec and generate-plan prompts and templates accordingly
- Update plan-reviewer to reject plans with non-verifiable AC

**Size:** medium. Unlocks #3.

---

## 3. Add `verify-task` subagent (replaces orchestrator self-audit)

**Problem:** Step 10 asks the orchestrator to verify output against AC inline. Doesn't scale, drifts silently.

**Scope:**
- New subagent prompt: receives task spec + produced files + AC only; returns yes/no + evidence per criterion
- Per-task (not per-wave): dispatched in parallel with coders or pipelined as each coder returns `DONE`
- Replace Step 10's manual verification with verifier dispatch
- Independent of coder (fresh context)

**Depends on:** #2 (verifiable AC format)
**Size:** medium

## 8. Tighten scout-brief contract

**Problem:** Generate-plan verifies the scout brief file exists and passes the path to the planner, but the planner is not required to read it. "Best non-obvious context" is on the honor system.

**Scope:** decide one of:
- **(A)** Make planner required to read the brief — enforce in planner prompt, validate with reviewer check ("did the plan incorporate brief findings?")
- **(B)** Drop the field entirely — remove `Scout brief:` line from spec/plan preamble, simplify define-spec and generate-plan accordingly

Recommend A if scout briefs are load-bearing in practice, B if they are aspirational.

**Size:** small (either direction)
